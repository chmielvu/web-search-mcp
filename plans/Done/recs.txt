6. Support partial results and cancel slow providers

Agents often need partial results quickly. The research findings note that AgentWebSearch-MCP returns partial results from fast providers while slower providers continue in the background. Implement a mechanism to return partial results after a short timeout and optionally stream additional results as they arrive. This improves user experience and reduces latency.

2. Integrate robust extraction services

The current get_content relies on trafilatura and a headless browser. To improve extraction quality:

Use Firecrawl extraction: Firecrawl’s API can return clean markdown, JSON and screenshots. It includes JS execution and automatic deobfuscation, outperforming generic headless browsers for dynamic pages. The research suggests Firecrawl is highly rated for agentic RAG tasks.
Add Jina Reader or Crawl4AI: Jina Reader (previously Jina AI Reader) quickly converts URLs into markdown for LLMs and has a generous free tier. Crawl4AI is an open‑source crawler that produces LLM‑friendly markdown and can run locally. These tools can act as alternative extraction strategies.
Implement an extraction fallback chain: Adopt the fallback chain recommended in the research findings (e.g., Firecrawl → Linkup → Tavily → Exa → You.com). If one extraction fails or returns low‑quality content, the next extractor can be invoked.
3. Enhance query rewriting and routing
Implement query classification: Use heuristics or a small language model to classify queries into categories (code/docs, product, news, multi-hop research, community). Map each category to a provider set and expansion strategy. For example, code queries could route to Exa and docs connectors; product research queries could route to Brave, Tavily and community sources.
Adopt structured expansion templates: Follow the best practices plan by always including the original query and adding complementary variants (documentation search, community opinions, comparative queries). Use templates that ensure synonyms, version keywords, and site filters are preserved; adjust expansion length based on query complexity and model size.
Temperature and multi‑hop control: Introduce heuristics to adjust LLM temperature for query rewriting (conservative vs. exploratory) and support multi‑hop detection to determine whether to break complex queries into sequential sub‑queries.

News/business connectors: Integrate providers such as NewsCatcher/CatchAll, Linkup and Valyu to gather current news and business insights. Agents can then answer queries about company updates or market movements.

The MCP implements an LLM‑based query rewriter and signals providers when rewriting is necessary. The Best Practices plan emphasises conservative rewriting, always including the original query, and generating complementary expansions rather than replacements. The plan also recommends classification by query type (e.g., code, product, news, multi‑hop) and generating specific expansions accordingly. Currently the server exposes some query guidance (e.g., instructs the agent to include site filters), but classification and routing are not yet implemented. The search orchestrator calls providers sequentially (free then paid) rather than routing per query class or using dynamic budgets.

Extraction and content pipeline

The get_content tool uses trafilatura for HTML extraction and falls back to a headless browser; summarisation is performed with simple heuristics. Trafilatura works well for many pages but struggles with dynamic or login‑gated sites. The repository does not integrate more robust extraction services like Firecrawl, Jina Reader, Crawl4AI, Spider.cloud or Apify. There is no explicit provider‑chain fallback for extraction (e.g., Firecrawl → Linkup → Tavily) as recommended in the research findings.

10. Encourage agent‑aware retrieval patterns

Agents must be trained to use search tools judiciously. Provide clear guidelines in the MCP documentation on when to search, how to choose providers, how to read rewrite metadata and how to avoid repeated queries. Offer sample prompts and heuristics for common tasks. This ensures that the tools are used effectively.
